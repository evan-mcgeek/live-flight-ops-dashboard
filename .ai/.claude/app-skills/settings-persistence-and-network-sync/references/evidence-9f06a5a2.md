The app needs a Settings slider for update interval (values {1,2,5,10,30,60,120}). The pattern implemented: (1) Persist locally via SettingsRepository (SharedPreferences) so the preference survives app restart; (2) POST to POST /aircraft/live-interval with X-Live-Interval-Seconds header when the slider changes, to notify the backend if the app is in SignalR mode (where it never makes REST polls); (3) Include the header on every GET /aircraft poll in Standard mode, so the backend sees the current preference and updates its shared state accordingly — this handles eventual consistency when server state changes elsewhere (other clients, server restart).
